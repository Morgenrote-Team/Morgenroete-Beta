#Vanilla project costs were often too high compared to Morgenröte projects.
#One problem in particular is the linear rise of project costs with vanilla cost modifiers.
#To reduce this problem, smaller projects that focus on one building should have a maximum cost

##### Normal Project Costs #####
#Normal Projects are projects that affect entire countries (e.g. Irrigation system project).
#These projects should not have upper limits for costs.

##### Local Project Costs #####
#Local Projects are projects that improve one building or similar.
#Local Projects take between 36 and 72 points to finish. The base rate is one point but it can be way faster.
#The absolute minimum would be 1 month for geological improvements and 3 months for other projects.
#Therefore I used a relatively flat rise (2/5 of vanilla) but with a minimum and maximum cost.

#The base cost is equivalent to the cost with 1 Million GDP.
#The maximum cost is equivalent to the cost with 100 Million GDP.

### Tiny Project ###
# Scope = Country
mr_project_cost_tiny_value = {
	value = gdp
	divide = 100000
	round = yes
	multiply = 0.3

	min = 10
}
mr_local_project_cost_tiny_value = {
	value = gdp
	pow = 0.5
	divide = 500
	round = yes
	multiply = 10

	min = 10
}
